The midterm project is a chance for you to give a popular but outdated website a fresh coat of paint using HTML & CSS!

Overview

Your task is to redesign and code the website for Meskie's Ethiopia Restaurant. It should have at least two distinct "views" — a home page and a menu page. Everything else is up to you!

This means that you have the creative freedom to add or remove content, change up the layouts to your liking, and customize the visual design. Feel free to include only a subset of the menu, change the theme of the restaurant completely, or use other pictures you find online — this project isn't meant to be a complete, functional redesign of their website. Read this whole page for complete details on grading, deadlines, and deliverables.

Reminder: While coding up your website, it is essential to link multiple pages properly so the audience can easily navigate the website by following links on your webpages. If you're unsure how to, ask us!

Why Meskie's Ethiopia Restaurant?

Meskie's Ethiopia Restaurant is a great restaurant, but their website is a little outdated. It suffers from some design problems we've explored in class — poor typography, improper spacing, color usage, etc. We want to be able to redesign this website such that its appearance advertise their restaurant well and attract new customers.

Visual Design (8 points)

Here are some things to consider as you design your site.

  • Color scheme: Are colors too bright or too dark? Is there enough contrast between the text color and background color?
  • Typography: Is the font readable? Are different pieces of text appropriately sized?
  • Visual hierarchy/Layout: Is content organized logically? Is there sufficient spacing?
  • Overall: Do your visual elements convey the mood you want to create on the site? Do your design choices outline visual hierarchy? Show your friends/roommates/classmates - what do they think?

User Experience (8 points)

For UX, think broad, about purpose. Here are some user experience considerations to think about

  • Does the user have a consistent experience across the whole site?
  • Is navigation clear and accessible?
  • Is content (menus, location, contact info, etc) easy to find?
  • Does this website make me want to go eat here?

Note on Functionality: We understand that we haven’t covered things like how to add a Google Maps widget, functioning buttons, or animations to your site. For more advanced functionality like that, we don’t expect a full implementation. For example, an image of a map would be sufficient.

Write-up (3 points)

Be concise about your design choices!

Attach a link of your completed write-up (a Google Doc) in your Google Form submission (more instructions below).

Note on Images: You can feel free to use any images you think would be appropriate for an hot dog restaurant's website. However, if you use images that you found on the internet, you must cite your sources (a link to the website you found it from is fine) at the end of your write-up.

A great site we recommend for finding stock photos is unsplash.com.

Recommended Workflow

The general steps that we recommend to complete this project are as follows:

  1. Look at Meskie's Ethiopia Restaurant's website. What do they do well and what can be improved? Decide on the content of each page.
  2. Create the general page structure with a wireframe. Draw by hand! Re-iterate and decide what works before moving on.
  3. Mock up both pages, including content and images in Figma. Play with the color scheme and fonts and decide on a consistent theme. The more developed your mockups are, the easier the coding will be! As you do this, think about the design decisions you’re making, which you’ll need to talk about in the write-up.
  4. Attend any TA or instructor’s office hours for your project check-in.
  5. Finalize the design, incorporating the TA’s feedback. This is also a good time to start the write-up.
  6. Create the HTML skeletons and CSS that would be used across every page - for example, the navigation links would have consistent CSS on every page of the site.
  7. Now that you have a “base,” you can more easily finish up the remaining pages.
  8. Check that the whole site has a cohesive look and feel.
  9. Complete the write-up and submit!
